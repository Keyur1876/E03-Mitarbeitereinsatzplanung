== Use-Case: UC02: Projekt bearbeiten


=== Kurzbeschreibung

Dieser Use-Case beschreibt Stammdaten von Projekten.

=== Kurzbeschreibung der Akteure

* Manager
* Administrator

=== Vorbedingungen
//Vorbedingungen müssen erfüllt, damit der Use Case beginnen kann, z.B. Benutzer ist angemeldet, Warenkorb ist nicht leer...

* Benutzer ist erfolgreich angemeldet.

=== Standardablauf (Basic Flow)
//Der Standardablauf definiert die Schritte für den Erfolgsfall ("Happy Path")

. Der Use Case beginnt, wenn der User auf Projekten von Kalendar auswählen.
. Projektname und Projektbeschreibung wird angezeigt.
. Auslastung von Projekt wird automatisch gerechnet.
. Kalendar mit aktuellen Tag wird angezeigt.
. Gesamtdatem von Task (Summe von Task, fertige Task, bearbeitende Task)
. Tabelle zur Darstellung von Task zu bearbeiten:
* Taskname.
* Fälligkeitsdatum zur Plannung.
* Status zur Tracking mit Combo Box (In Bearbeitung, Fertig, Überfällig).
* Task hinzufügen.
. Tabelle zur Darstellung des kommenden Termin:
* Termin bearbeiten.
* Termin hinzufügen.
. Zuordnung Teamitgliedern in dieser Projekt:
* Teammiglieder hinzufügen.
* Teammiglieder löschen.
. Der Use Case ist abgeschlossen.

=== Alternative Abläufe

* Wenn User die Software nicht benutzen, wird der System automatisch abgemeldet.
* Wenn Internet-Zugang abgebrochen wird, wurden Änderungen nicht gespeichert.
